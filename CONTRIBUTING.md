# Contributing to GopherNotebook

First off, thank you for considering contributing to GopherNotebook! It's people like you that make it a great tool for the community.

## How Can I Contribute?

### Reporting Bugs
- Use the [GitHub Issue Tracker](https://github.com/RobinMillford/GopherNotebook/issues) to report bugs.
- Provide a clear, descriptive title.
- Include steps to reproduce the issue.

### Suggesting Enhancements
- Open an issue with the "enhancement" tag.
- Explain why the feature would be useful and how it should work.

### Pull Requests
- Fork the repository.
- Create a new branch for your feature or fix.
- Ensure your code follows the existing style (Go/Next.js).
- Submit a Pull Request to the `main` branch.
- **Note:** All PRs will be reviewed by the lead maintainer (@RobinMillford).

## Project Ownership & Attribution
GopherNotebook is an open-source project maintained by **Robin Millford**. By contributing, you agree that your contributions will be licensed under the project's [MIT License](LICENSE).

Development and main design decisions are led by the primary author. Contributors will be credited in the repository's contributor list, but the core project remains the intellectual property and portfolio highlight of the founder.

## Style Guide
- **Go**: Follow `gofmt` and standard Go idioms.
- **Frontend**: Use functional components with Tailwind CSS (shadcn/ui style).
- **Commit Messages**: Use [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) (e.g., `feat:`, `fix:`, `docs:`).

Happy coding!
